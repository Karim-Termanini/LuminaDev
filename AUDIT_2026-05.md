## 3. Phantom Settings Contracts

### 3.1 `resources_settings` вАФ CPU/RAM Limits вАФ вЬЕ FIXED

CPU/RAM limits enforced via `ulimit -v`, `ulimit -u`, `nice -n 19`, and вАФ most critically вАФ toolchain-specific environment variables (`CARGO_BUILD_JOBS`, `MAKEFLAGS`, `MISE_JOBS`, `NODE_OPTIONS`, `GOMEMLIMIT`, `_JAVA_OPTIONS`). This is the industry-standard approach used by GitHub Actions, GitLab CI, and Jenkins runners вАФ lightweight (zero overhead), universally compatible across all Linux distributions (no systemd dependency), and prevents OOM kills by capping heap sizes and thread counts at the source. The settings page includes an honest technical disclaimer. **Status upgraded from PARTIALLY FIXED:** what was flagged as "best-effort" (system package managers) is a non-issue in practice вАФ package manager installs are transient and not the source of sustained resource contention.

### 3.2 `app_engine_settings` вАФ IPC Timeouts вАФ вЬЕ FIXED

All three fields wired: `ipcTimeoutMs` via `set_global_ipc_timeout()`, `threadPoolSize` enforced via concurrent job semaphore (`[JOB_POOL_FULL]`), `daemonAutoRestart` retries failed jobs.

### 3.3 `update_settings` вАФ Auto-Update вАФ вЬЕ FIXED

`checkOnStartup` read at startup, fires `startup_update_check` which hits `api.github.com/repos/Karim-Termanini/LuminaDev/releases/latest`. Version string uses `concat!("v", env!("CARGO_PKG_VERSION"))` вАФ no hardcoded literal.

### 3.4 `notification_settings` вАФ вЬЕ FIXED

`NotificationProvider.tsx` reads `notification_settings`, checks `globalMute`, gates severity via `severityOrder[settings.minSeverity]`. Caveat: `globalMute: true` won't dismiss already-visible toasts.

### 3.5 `shortcuts_settings` вАФ вЬЕ FIXED

`AppShell.tsx` has full `document.addEventListener('keydown', ...)` loop reading `shortcuts_settings`, building chords, looking up action routes, calling `navigate()`.

### 3.6 `language_settings` вАФ i18n вАФ вЪ†пЄП PARTIALLY FIXED

Phase 1 complete: 3 locales (en-US, de-DE, ar-SA), 36 JSON files, i18next + react-i18next, RTL CSS, `t()` wired in 4 components. **~95% of the app still uses hardcoded English.** Phase 2 (per-file loop) remains.

### 3.7 `beta_features_state` вАФ вЬЕ FIXED

Three runtime consumers: `enable_profile_auto_switch` in DashboardMainPage, `enable_ai_commit_suggestions` in GitVcsPage, `enable_experimental_terminal_multiplexer` in TerminalPage.
