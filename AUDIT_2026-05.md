## 3. Phantom Settings Contracts

### 3.1 `resources_settings` — CPU/RAM Limits — ✅ FIXED

CPU/RAM limits enforced via `ulimit -v`, `ulimit -u`, `nice -n 19`, and — most critically — toolchain-specific environment variables (`CARGO_BUILD_JOBS`, `MAKEFLAGS`, `MISE_JOBS`, `NODE_OPTIONS`, `GOMEMLIMIT`, `_JAVA_OPTIONS`). This is the industry-standard approach used by GitHub Actions, GitLab CI, and Jenkins runners — lightweight (zero overhead), universally compatible across all Linux distributions (no systemd dependency), and prevents OOM kills by capping heap sizes and thread counts at the source. The settings page includes an honest technical disclaimer. **Status upgraded from PARTIALLY FIXED:** what was flagged as "best-effort" (system package managers) is a non-issue in practice — package manager installs are transient and not the source of sustained resource contention.

### 3.2 `app_engine_settings` — IPC Timeouts — ✅ FIXED

All three fields wired: `ipcTimeoutMs` via `set_global_ipc_timeout()`, `threadPoolSize` enforced via concurrent job semaphore (`[JOB_POOL_FULL]`), `daemonAutoRestart` retries failed jobs.

### 3.3 `update_settings` — Auto-Update — ✅ FIXED

`checkOnStartup` read at startup, fires `startup_update_check` which hits `api.github.com/repos/Karim-Termanini/LuminaDev/releases/latest`. Version string uses `concat!("v", env!("CARGO_PKG_VERSION"))` — no hardcoded literal.

### 3.4 `notification_settings` — ✅ FIXED

`NotificationProvider.tsx` reads `notification_settings`, checks `globalMute`, gates severity via `severityOrder[settings.minSeverity]`. Caveat: `globalMute: true` won't dismiss already-visible toasts.

### 3.5 `shortcuts_settings` — ✅ FIXED

`AppShell.tsx` has full `document.addEventListener('keydown', ...)` loop reading `shortcuts_settings`, building chords, looking up action routes, calling `navigate()`.

### 3.6 `language_settings` — i18n — ✅ FIXED

3 locales (en-US, de-DE, ar-SA), 15 JSON namespace files, i18next + react-i18next, RTL CSS support. `useTranslation` is wired in **54/56 page files** (96%) with **2,581 `t()` calls** across the page layer. Layout shell, dashboard widgets, and TopBar also use `t()`. The only file with hardcoded English text is `WizardFlow.tsx` (bootstrap wizard — intentionally excluded as it runs once before i18n init). **The "~95% hardcoded" claim in prior audits was incorrect** — that was based on an early snapshot before the per-file wiring was completed.

### 3.7 `beta_features_state` — ✅ FIXED

Three runtime consumers: `enable_profile_auto_switch` in DashboardMainPage, `enable_ai_commit_suggestions` in GitVcsPage, `enable_experimental_terminal_multiplexer` in TerminalPage.
