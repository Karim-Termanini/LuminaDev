# KeelDev — naming map

**Product name (user-facing):** **KeelDev**

Docs (`README.md`, `phasesPlan.md`, `MASTER_PLAN.md`) and new UI copy use **KeelDev**. Legacy strings may still say LuminaDev in Rust HTTP clients or generated scaffolds until migrated.

## Why four names appear

| Name | Where | Role | Change? |
| --- | --- | --- | --- |
| **KeelDev** | Docs, window title, `productName` | Display / marketing | **Canonical** for new work |
| **keel-dev** | Root `package.json` `name` | npm workspace slug (kebab-case) | Keep — not user-visible |
| **@linux-dev-home/shared** | `packages/shared`, imports | npm scope from original “Linux Dev Home” codename | **Frozen** — rename breaks workspace + all imports |
| **LinuxDevHome** | `io.github.karimodora.LinuxDevHome.*` icons, `.desktop` basename | Freedesktop app ID / icon theme path | **Frozen** — changing breaks installed `.desktop` + icon cache |
| **LuminaDev** | GitHub repo URL, clone dir, Tauri `identifier` (`io.github.karimodora.luminadev`), some Rust `User-Agent` strings | Repo + bundle identity from prior rebrand | Repo/identifier frozen; HTTP UA + scaffolds → KeelDev over time |
| **LuminaDev** (directory) | `~/Documents/GitHub/LuminaDev/` in maintainer docs | Physical clone path on disk | Matches GitHub repo name until repo is renamed |

## Frozen identifiers (do not rename without migration plan)

- **Tauri bundle ID:** `io.github.karimodora.luminadev` — new ID = side-by-side install, lost app data path.
- **Icon / desktop files:** `data/icons/.../io.github.karimodora.LinuxDevHome.*`, `desktopName` in `apps/desktop/package.json`.
- **npm package:** `@linux-dev-home/shared` and workspace filter names.
- **GitHub remote:** `Karim-Termanini/LuminaDev` until the repository is renamed on GitHub.

## Aligned (2026-06-19)

- `tauri.conf.json` — `productName` and window `title` → **KeelDev**
- This document — single source for contributors
